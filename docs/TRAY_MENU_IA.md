# Tray / 原生菜单 IA 草案

## 目标

- 托盘承担“轻运营入口”，主窗口承担“重配置与批处理”。
- 在不新增第二套账号模型前提下，支持快速切号与状态感知。
- 与主窗口、浮动账号卡片保持事件级联动。

## 菜单分层

1. 固定入口层
   - `打开主界面`
   - `打开账号与资产库`
   - `托盘范围：...`（只读）
2. Provider 层
   - Provider 切换（单选）
3. 账号快照层
   - `账号概览：总 X · 当前 Y · 需关注 Z`
   - 当前账号快照列表（只读）
4. 快速切号层
   - 按平台分组（`codex/gemini/...`）
   - 每平台展示有限条（避免菜单过长）
   - 超出部分提示“请到主窗口处理”
5. 系统动作层
   - 刷新余额
   - 退出

## 交互与事件

- 托盘切 Provider:
  - 更新托盘勾选状态
  - 广播 `data:changed`（domain=`providers`）
  - 主窗口监听后刷新
- 托盘快速切号:
  - 调用现有注入链路
  - 广播 `data:changed`（domain=`ide_accounts`）
  - 广播 `floating.account.changed`（供浮窗联动）
  - 托盘菜单即时重建
- 托盘打开账号页:
  - 主窗口 `show + focus`
  - 发送 `navigate_to_page=accounts`

## 异常兜底

- 菜单项解析失败：通知“托盘菜单项格式异常”
- 账号已删除：通知“目标账号不存在，托盘已刷新”
- 注入失败：通知失败原因（沿用现有错误链路）
- 当前范围无可切换账号：展示只读空态项

## 测试建议

- 纯函数测试：
  - 菜单 ID 解析（有效/无效）
  - 账号概览文案格式
- 手工联调：
  - 托盘切号后主窗口资产页高亮刷新
  - 托盘切号后浮窗账号状态同步
  - 托盘范围切换后菜单内容收敛
